Windows API - Lab part

Pong

  • Sample application
  • Description:
    • The application contains of three windows: the main application window, the ball and the paddle
    • Paddle movement is done by moving the mouse
    • Ball bounce of main window walls and paddle
    • When the ball misses the paddle, the game ends - the paddle and ball stop moving
    • Attention: Do not use GDI in Lab part. All objects are WinAPI windows.
  • Functionality:
    • Main window:
      • Colour: light green, transparency 20%
      • Has changed title
      • Size 500x350, fixed size, but can be moved around the screen
      • At startup, the application window is centred on the screen
    • Ball:
      • Round, red colour
      • Position updates every 50 ms (use timers)
    • Paddle:
      • Has the colour of the active title bar
      • The mouse cursor moves the centre of the paddle
      • A paddle is moving at the right edge of the main window
  • Additional requirements:
    • Only one window is visible on the taskbar
    • Key shortcut Alt+F4 close the application
    • Key shortcut Alt+Space still works as in the default application
  • Tips:
    • SetLayeredWindowAttributes(), WS_EX_LAYERED, WS_MAXIMIZEBOX
    • WS_CHILD, WS_CLIPSIBLINGS, WS_CLIPCHILDREN
    • SetWindowRgn(), GetWindowRect(), MoveWindow()
    • Window regions: 
      HRGN region=CreateEllipticRgn(0,0,10,10);
SetWindowRgn(hWnd,region,true);
    • SetTimer(), WM_TIMER
    • GetCursorPos(), WM_MOUSEMOVE
    • ClientToScreen(), ScreenToClient()
    • InvalidateRect()
  • Note: The application should behave like an example app in all doubtful and untold aspects.
  • Scoring:
    • Displaying the main window according to the task description: 3 points
    • Displaying paddle window (as child window): 2 points
    • Proper displaying ball window (no movement): 2 points
    • Ball movement (proper timer usage): 2 points
    • Paddle movement: 2 points
    • The ball stops at missing paddle: 1 point
    • Note: there are no points for partial functionality

Windows API - Home part

  • Sample application
  • Task description:
    • Main menu options
    • Counter of ball bouncing with paddle
    • Ball has trail
  • Menu:
    • Proper menu shortcuts (as in the sample application)
    • Game reset option
    • Changing background colour using dialog window
    • Loading bitmap as window background
    • The bitmap can be tiled or stretched
    • Menu marks update on the change
  • Counters:
    • Has the colour of the negated background
    • Left counter count missing ball
    • Right counter count paddle hits
    • Counters are centred in 1/4 and 3/4 of window width, respectively
  • Ball trail:
    • It is built with circled windows
    • Its radius changes with time
    • When it disappears, it should be removed (not hidden)
  • Tips:
    • CreateCompatibleBitmap, BitBlt
    • CreatePatternBrush, SetStretchBltMode
    • TextOut, DrawText, SetTextAlign
    • ChooseColor, GetOpenFileName
    • CheckMenuItem, EnableMenuItem
  • Note: In all doubtful and untold aspects application should behave like an example app.
  • Scoring:
    • Main menu and game reset: 1 point
    • Main menu shortcuts: 1 point
    • Working counters: 1 point
    • Changing background: 2 points
    • Loading bitmap: 3 points
    • Ball trail: 2 points
    • Changing bitmap mode: 2 points
    • Note: To be able to get points from Home part, all requirements from Lab part must be fulfilled
